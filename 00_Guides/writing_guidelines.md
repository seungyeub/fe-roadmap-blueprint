# ✍️ Writing Guidelines – 문서 작성 규칙

> `fe-roadmap-blueprint` 안의 모든 md 파일은
> 가능한 한 **비슷한 톤과 구조**를 유지하도록 합니다.

---

## 1. 공통 제목/구조

각 토픽 파일(예: `02_JS/async.md`)은 기본적으로 아래 구조를 따릅니다.

```markdown
# 📌 제목 (예: JS 비동기 처리 – 이벤트 루프, Promise, async/await)

> 이 문서의 목표를 한두 줄로 요약합니다.

---

## 1. 개념 정의

## 2. 왜 중요한가? (언제 쓰이고, 어떤 문제가 해결되는가)

## 3. 동작 원리 / 내부 구조

## 4. 코드 예제

## 5. 자주 하는 실수 & 주의할 점

## 6. 면접 질문 예시

## 7. 추가로 보면 좋은 문서/키워드
```

필요하면 섹션은 자유롭게 추가/삭제하되,
1. 정의 → 2. 중요성 → 3. 원리 → 4. 예제 → 5. 실수/면접 흐름은 유지합니다.

## 2. 마크다운 스타일

- 제목은 `#`, `##`, `###` 순으로 사용
- 리스트는 `-` 또는 번호 리스트 `1.` 사용
- 코드 블럭은 언어를 명시

```
(```js)
function example() {
  console.log('hello');
}
(```)
```

- 중요한 단어는 `**굵게**`

---

## 3. 이모지 사용 규칙 (선택)

- 문서 제목 앞: 해당 주제를 잘 표현하는 이모지 1개
  - 예: 네트워크 → 🌐, HTTP → 📡, JS 기본 → 🧩, TS → 🧷, React → ⚛️
- 섹션 제목:
  - 개요/정의 → 📍
  - 중요성 → ⭐
  - 원리 → ⚙️
  - 예제 → 💡
  - 실수/주의 → ❗
  - 면접 질문 → 🎯

이모지는 필수는 아니지만, 시각적인 구분에 도움이 됩니다.

---

## 4. 코드 예제 작성 규칙

- **최소 1개 이상**의 코드 예제를 넣는 것을 기본 목표로 합니다.
- 예제는
  - “최소한의 코드로 개념이 드러나게”
  - 실제로 실행해 볼 수 있도록
- 가능하면
  - 나쁜 예제 → 좋은 예제
  를 비교해서 설명합니다.

---

## 5. 면접 질문 섹션

각 문서에 **간단한 면접 질문**을 넣어 두면 좋습니다.

예:

```markdown
## 6. 면접 질문 예시

1. 이벤트 루프(Event Loop)가 무엇이고, 어떤 순서로 동작하나요?
2. Promise와 async/await의 차이점을 설명해 보세요.
3. 마이크로태스크(microtask)와 매크로태스크(macrotask)는 무엇인가요?
```

---

## 6. 링크 & 참고자료
- 외부 링크를 넣을 때는
  - 공식 문서
  - MDN, React 공식 문서, TypeScript 핸드북 등
- 상단이나 하단에 `## 7. 추가로 보면 좋은 문서/키워드`로 모아둡니다.
```
## 7. 추가로 보면 좋은 문서/키워드

- MDN – Promise
- MDN – async function
- 검색 키워드: "javascript event loop", "call stack, task queue"
```

---

## 7. 깊이 조절
- 처음 작성할 때는
  - 정의 + 간단 예제 + 면접 질문 2~3개 정도만 적어도 충분.
- 나중에 복습할 때마다
  - 그림 추가
  - 예제 코드 늘리기
  - "실수/주의점" 섹션에 경험 추가

이런 식으로 점진적으로 깊이를 늘려가는 방식을 기본으로 합니다.
