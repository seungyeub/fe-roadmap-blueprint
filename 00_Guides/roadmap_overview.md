# 🧭 FE Roadmap Blueprint – Overview

> 이 레포지토리는 **프론트엔드 개발자가 되기 위해 공부해야 할 이론·개념·패턴**을
> 체계적으로 정리해 두는 “지식 설계도(BluePrint)”입니다.

- **fe-roadmap-blueprint**: 개념/이론/패턴 정리 (**지식 저장소**)
- **fe-roadmap-progress**: 학습 일정, 일일/주간 로그, 진행률 관리 (**진행 기록**)
- **fe-roadmap-lab**: 실험/미니 프로젝트, 코드 연습 (**실습 공간**)

---

## 1. 이 레포지토리의 역할

1. **공부해야 할 영역을 빠짐없이 나열**합니다.
2. 각 주제별로
   - 개념 설명
   - 그림/도식
   - 코드 예제
   - 흔한 면접 질문
   - 추가 참고 링크
   를 하나의 문서 안에 정리합니다.
3. 이 레포를 보면
   **“지금까지 무엇을 이해하고 있는지 / 앞으로 무엇을 채워야 하는지”**가 한눈에 보이게 하는 것이 목표입니다.

---

## 2. 상위 폴더 구조 (임시)

```bash
fe-roadmap-blueprint/
├── 00_Guides/
├── 01_CS_Basics/
├── 02_JS/
├── 03_TS/
├── 04_React/
├── 05_State_Management/
├── 06_Styling/
├── 07_Tooling_Testing/
├── 98_Projects/
└── 99_Interview/
```

각 폴더의 의도
- 01_CS_Basics
  - 네트워크, HTTP, 브라우저 동작 원리, OS/메모리 등
  - FE 면접에서 자주 나오는 CS 기초 개념

- 02_JS
  - JS 기본 문법부터 스코프/클로저/this/비동기/프로토타입 등
  - “왜 이렇게 동작하는지”를 설명하는 본진

- 03_TS
  - TS 타입 시스템, 제네릭, 유틸리티 타입, TS 환경 설정 등

- 04_React
  - JSX, 컴포넌트 구조, Hooks, 렌더링 최적화, Suspense 등

- 05_State_Management
  - Redux, Redux Toolkit, Recoil, Zustand 등 상태 관리 도구

- 06_Styling
  - CSS, SCSS, CSS-in-JS, 디자인 시스템, 반응형 레이아웃 등

- 07_Tooling_Testing
  - 빌드 툴(Vite, Webpack), 번들링, Lint/Format, Jest/RTL 등

- 98_Projects
  - 프로젝트 설계 아이디어, 아키텍처 정리, 폴더 구조 패턴

- 99_Interview
  - 면접 질문 리스트, 예상 답변, 꼬리질문 정리
